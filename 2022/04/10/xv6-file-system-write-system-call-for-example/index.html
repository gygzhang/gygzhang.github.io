lab 大文件支持

该函数也是lab之一需要大改的函数之一，对应的任务需要我们实现更大的文件支持，在原始的实现中，文件的大小为11个block+256个block=267个block，这是由inode的定义决定的：

NDIRECT的值是11，代表着11个block，额外的1代表一个指针，指向了另一个block，而block中的每个元素为一个指针，分别指向一个block，一个block的大小/指针的大小=256，所以说文件的最大的大小为267block。我们的任务是使最大文件大小变大，既然使用了指针，所以我可以考虑再次使用指针，使用一个二级指针来实现，具体思想就是将inode中的addrs的大小变成NDIRECT+2， 多出的1是一个二级指针，指向一个指针block，而这个指针block又指向一个指针block，这一级的指针block的每个元素都指向一个指针的数据block。示意图如下：

为什么在系统crash之后，文件系统还能恢复？

在系统启动的时候，会执行recover_from_log()这个函数，该函数的定义如下：

该函数首先执行read_head来获取log区中刚刚由write_head写入的磁盘号，将其读入内存中的log.lh.block中。然后再执行install_trans,这次不同于commit函数中的install_trans，本次install_trans的参数为1，表示执行恢复功能。需要特别注意的是，这一次执行install_trans的时候，由于断电，内存中已经没有相关数据了，这个时候，就只能去磁盘中的log区中读取(由write_log写入)。由于内存数据丢失，所以每一个buf对应的refcnt也不见了，所以不应该执行bunpin操作。这正是install_trans传入1时的操作。
